# Zhenya’s English Space

Персональный статический сайт ученицы Жени для занятий по учебнику **English File, 4th edition Elementary**.

Параметры проекта:

- уровень: A1;
- интерфейс и объяснения: около 90% на русском;
- английский используется в учебных словах, примерах, текстах и проверяемых заданиях;
- авторизация не используется;
- Supabase хранит только прогресс и технические данные уведомлений;
- материалы находятся в GitHub.

## 1. Где изменить данные ученика

Откройте `config.js`:

```js
student: {
  id: "zhenya",
  nameRu: "Женя",
  nameEn: "Zhenya",
  level: "A1",
  textbook: "English File",
  textbookEdition: "4th edition Elementary"
}
```

Для другого ученика обязательно создавайте отдельный проект и отдельные RLS-политики. Не меняйте только имя в уже используемой базе.

## 2. Настройка Supabase

В `config.js` заполните:

```js
supabase: {
  url: "https://PROJECT-REF.supabase.co",
  anonKey: "PUBLIC_ANON_OR_PUBLISHABLE_KEY"
}
```

В клиентском коде допустимы только Project URL и публичный ключ. Никогда не добавляйте приватный серверный ключ, Telegram Bot Token или Supabase Access Token.

Откройте Supabase → **SQL Editor** и выполните:

1. `supabase/schema.sql`;
2. `supabase/verify.sql`.

SQL создаёт ровно шесть таблиц:

1. `homework_progress`;
2. `vocabulary_progress`;
3. `vocabulary_topic_progress`;
4. `grammar_progress`;
5. `telegram_recipients`;
6. `material_publications`.

Таблица `student_progress` не используется: это наследие старой архитектуры.

## 3. Как сайт работает без авторизации

На сайте нет регистрации, email, пароля и страницы входа. Supabase Client работает с ролью `anon`, а RLS разрешает доступ только к строкам `student_id = 'zhenya'` в четырёх таблицах прогресса.

Ограничение: без личной авторизации любой человек, который знает адрес персонального сайта, технически может открыть интерфейс этой ученицы. Поэтому адрес не следует публиковать широко. Секреты и серверные таблицы при этом остаются недоступными браузеру.

## 4. Что хранится в Supabase

Supabase хранит:

- черновики и единственную финальную сдачу домашней работы;
- ответы, баллы, даты проверки и отправки;
- статус доставки Telegram-отчёта;
- статусы слов `new`, `known`, `difficult`;
- историю словарных тестов;
- количество попыток и лучший результат по грамматике;
- серверный маршрут Telegram;
- журнал уведомлений о новых материалах.

Supabase не хранит:

- тексты уроков;
- правильные ответы как источник материала;
- словарные слова и переводы;
- грамматические объяснения;
- изображения и MP3;
- данные учебника.

## 5. LocalStorage и синхронизация

Сайт сначала сохраняет прогресс на устройстве, затем синхронизирует его с Supabase. Используются ключи:

```text
english_space_zhenya_homework
english_space_zhenya_vocabulary
english_space_zhenya_grammar
english_space_zhenya_sync_queue
```

Если Supabase временно недоступен, локальные данные не удаляются. После восстановления интернета сайт повторяет синхронизацию.

## 6. Как добавить домашнюю работу

Создайте файл:

```text
data/lessons/lesson-1.json
```

Подробный формат находится в `data/lessons/README.md`.

Обычное добавление урока не требует изменения:

- `app.js`;
- `lesson-engine.js`;
- HTML;
- CSS;
- SQL.

Нумерация должна быть последовательной: `lesson-1.json`, `lesson-2.json` и далее. Сайт завершает поиск после трёх отсутствующих номеров подряд.

## 7. Как добавить словарь

Словарь хранится внутри JSON урока в объекте `vocabulary`. Каждому слову укажите:

- английское написание;
- русский перевод;
- транскрипцию по возможности;
- пример на английском;
- перевод примера.

Одинаковое слово учитывается один раз по нормализованному английскому ключу.

## 8. Произношение слов

Словарь использует Web Speech API браузера:

```text
language: en-GB
rate: 1
pitch: 1
volume: 1
```

Для слов не нужны отдельные MP3. Символ `/` в слове или фразе произносится как `or`.

## 9. Как добавить грамматику

Создайте:

```text
data/grammar/grammar-1.json
```

Подробный формат находится в `data/grammar/README.md`.

Для Жени правила и инструкции должны быть преимущественно на русском, потому что уровень A1 и параметр русского текста равен 90%. Английский остаётся в изучаемых конструкциях и упражнениях.

Каждая тема должна содержать пять заданий:

1. самое лёгкое;
2. лёгкое;
3. среднее;
4. повышенной сложности;
5. самое сложное.

Тема считается пройденной при результате от 80%. В Supabase сохраняются попытки и лучший результат; последние ответы дополнительно восстанавливаются из localStorage.

## 10. Аудиоупражнения

MP3 загружаются в:

```text
audio/lessons/
```

Рекомендуемое имя:

```text
audio/lessons/lesson-1-listening-1.mp3
```

Путь указывается в JSON урока.

## 11. Одноразовая сдача домашней работы

До отправки:

- ответы автоматически сохраняются как черновик;
- их можно менять;
- кнопка **Проверить ответы** показывает правильные ответы зелёным, неправильные красным;
- результат и подсветка восстанавливаются после перезагрузки.

После проверки появляется кнопка **Отправить учителю**. Перед отправкой показывается подтверждение.

После отправки:

- ответы фиксируются;
- работа считается завершённой при любом проценте правильных ответов;
- поля блокируются;
- повторное выполнение невозможно;
- отчёт отправляется через Edge Function;
- при ошибке появляется кнопка **Повторить отправку отчёта**.

Блокировка обеспечивается не только интерфейсом, но и SQL-триггером `prevent_homework_resubmission()`.

## 12. Telegram-отчёты по домашней работе

Одна Edge Function `notify-telegram` работает в двух режимах.

### Отчёт по домашней работе

Браузер передаёт только:

```json
{
  "action": "homework_report",
  "studentId": "zhenya",
  "lessonId": "lesson-1",
  "submissionId": "uuid"
}
```

Функция сама читает зафиксированный результат из Supabase. Баллы из браузера не принимаются. Режим отчёта не требует `x-notify-secret`, иначе сайт получит HTTP 401.

### Уведомление о новом материале

GitHub Actions использует:

```json
{
  "action": "material_published",
  "studentId": "zhenya",
  "materialType": "homework",
  "materialId": "lesson-1",
  "notificationVersion": 1,
  "payload": {}
}
```

Только этот режим требует секрет `NOTIFY_WEBHOOK_SECRET`.

## 13. Настройка Telegram

В GitHub → **Settings → Secrets and variables → Actions** добавьте:

### Secrets

- `SUPABASE_ACCESS_TOKEN`;
- `TELEGRAM_BOT_TOKEN`;
- `NOTIFY_WEBHOOK_SECRET`.

### Variables

- `SUPABASE_PROJECT_ID`;
- `SITE_BASE_URL`.

Реальный `chat_id` добавьте в таблицу `telegram_recipients` по инструкции `supabase/README.md`. Не храните его в публичном репозитории.

Запустите workflow:

```text
1 - Настройка Telegram
```

После изменения `supabase/functions/notify-telegram/index.ts` этот workflow нужно запускать повторно. В ответе функции должна быть версия:

```text
homework-reports-v1
```

## 14. Уведомления о новых материалах

Workflow `2 - Уведомить о новых материалах` запускается:

- автоматически после публикации JSON урока или грамматики;
- вручную с указанием пути к JSON.

Защита от повторных сообщений хранится в `material_publications`. Чтобы отправить обновлённое уведомление для того же материала, увеличьте:

```json
"notification": {
  "version": 2
}
```

## 15. Диагностическая страница

Откройте:

```text
telegram-report-test.html
```

Страница проверяет:

- `config.js`;
- доступ к Supabase;
- создание и чтение тестовой записи;
- Edge Function;
- HTTP-статус;
- `functionVersion`;
- фактическую отправку Telegram-отчёта.

Если появляется HTTP 401, значит в Supabase развёрнута старая версия функции или режим отчёта ошибочно требует секрет.

После успешной настройки удалите диагностическую страницу из публичного репозитория.

Тестовые строки можно удалить в SQL Editor:

```sql
delete from public.homework_progress
where student_id = 'zhenya'
  and lesson_id like 'telegram-report-test-%';
```

## 16. GitHub Pages

1. Загрузите содержимое папки проекта в корень репозитория.
2. Откройте **Settings → Pages**.
3. Выберите публикацию из ветки `main`, папка `/root`.
4. Сохраните настройки.
5. Добавьте опубликованный адрес в GitHub Variable `SITE_BASE_URL`.

Сайт нужно открывать через GitHub Pages или локальный HTTP-сервер. При открытии HTML двойным кликом браузер может запретить загрузку JSON через `fetch`.

## 17. Очистка кэша

После замены файлов выполните жёсткое обновление:

```text
Mac: Command + Shift + R
Windows: Ctrl + F5
```

Также можно открыть сайт в приватном окне.

## 18. Безопасность

Никогда не добавляйте в GitHub:

- Telegram Bot Token;
- Supabase Access Token;
- приватный серверный ключ;
- реальные пароли;
- `.env`;
- реальный `chat_id`.

Браузер не обращается к Telegram API напрямую. Все сообщения отправляет Edge Function.

## 19. Начальное состояние

Проект специально поставляется без фиктивных уроков, слов, грамматики и прогресса. На страницах показываются корректные пустые состояния:

- первое домашнее задание скоро появится;
- словарные тренажёры будут добавляться после уроков;
- грамматические темы пока не опубликованы;
- материалы по учебнику добавит преподаватель.
